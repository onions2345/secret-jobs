SECRET JOBS — EASY SETUP ON GITHUB (runs itself, free)
======================================================
Nothing runs on your computer or your cPanel hosting. GitHub's free
servers run the program on a schedule and show the website online.

WHAT YOU NEED
  - A free GitHub account: https://github.com
  - A free Google AI key:  https://aistudio.google.com/apikey

STEP 1 — MAKE A REPOSITORY
  Sign in to github.com. Click the + (top right) -> New repository.
  Name: secret-jobs   |   choose Public   |   click Create repository.

STEP 2 — UPLOAD THESE FILES
  On the new repo page click the link "uploading an existing file".
  Open this secret-jobs folder, select EVERYTHING inside it, and drag it
  into the GitHub upload box. Wait for it to finish, then click
  "Commit changes".

STEP 3 — ADD YOUR KEY (kept private)
  In the repo: Settings -> Secrets and variables -> Actions ->
  "New repository secret".
  Name:  GEMINI_API_KEY
  Secret: paste your Google AI key
  Click Add secret.

STEP 4 — TURN ON THE WEBSITE
  Settings -> Pages -> under "Source" choose "GitHub Actions". Save.

STEP 5 — RUN IT THE FIRST TIME
  Click the "Actions" tab -> click "Build Secret Jobs" on the left ->
  click "Run workflow" -> "Run workflow".
  Wait ~3-5 minutes. A green tick means it worked.

DONE
  Your site is live at the address shown in Settings -> Pages
  (looks like https://YOURNAME.github.io/secret-jobs ).
  From now on it rebuilds AUTOMATICALLY on a schedule and finds more
  jobs over time. You do nothing.

USE YOUR OWN DOMAIN (optional, later)
  config.yaml already has secretjobs.work.gd set. To point your domain
  at the GitHub site, add a DNS CNAME record:
     secretjobs.work.gd  ->  YOURNAME.github.io
  then Settings -> Pages -> Custom domain -> secretjobs.work.gd.

CHANGE WHAT IT LOOKS FOR (optional)
  Edit config.yaml on GitHub:
   - start_location, max_locations_per_run, categories
